面向对象设计与设计模式

发表于 2019-03-10 | 分类于 进阶

前言

从定义上来讲，设计模式（Design pattern）是一套被反复使用、多数人知晓的、经过分类编目的、代码设计经验的总结。
使用设计模式就是为了可重用代码、让代码更容易被他人理解、保证代码可靠性。
也正是由于设计模式本身就是在设计中经验的总结，所以要想把每一种设计模式融入到自己开发的设计中，没有经验的积累是很难实现的。菜鸟如我自然在运用设计模式的过程中倍感生涩，希望这次对设计模式的回顾总结过程，能加深对设计模式的理解吧。

面向对象设计（OOD）

在理解设计模式之前，首先需要对面向对象设计的几大特征和原则有所了解。在实际项目的开发过程中，为了提高代码的灵活性和可维护性，采用面向对象的方式，将现实世界的事物抽象成对象，把现实世界中的关系抽象成类。程序就成了包含各种独立而又互相调用的对象的组合。

面向对象三大基本特性

封装：封装就是把过程和数据给整合到一个实体中封闭起来，访问封装的数据只能通过内部定义好的对外界的接口。封装可以使代码模块化。

继承：继承是一种类的层次模型，允许和鼓励程序在原有类的基础上进行延伸，继承提高了代码的复用性

多态：多态的实质就是让一个接口实现多种方法，多态包括编译时多态（静态多态）和运行时多态（动态多态）。多态的使用让接口和实现分离开，改善了代码的组织结构，增强了代码的可读性和维护性。

面向对象中类的关系

在OOD中，类与类之间主要有6中关系模式，这六种关系具有不同的耦合度，具体如下所示：

  1. 依赖关系
  2. 泛化关系
  3. 关联关系
  4. 聚合关系
  5. 组合关系
  6. 实现关系

依赖关系

依赖关系(Dependency) 是一种使用关系，特定事物的改变有可能会影响到使用该事物的其他事物，在需要表示一个事物使用另一个事物时使用依赖关系。大多数情况下，依赖关系体现在某个类的方法使用另一个类的对象作为参数。

C#中的委托和事件

发表于 2019-03-10 | 分类于 基础

委托是一种引用类型，它是对具有特定参数列表和返回类型的方法的引用。在了解委托之前，首先要理解值类型和引用类型的概念。

值类型和引用类型

声明一个值类型变量，编译器会在栈上分配一个空间，这个空间对应着该值类型变量，空间里存储的就是该变量的值。引用类型的实例分配在堆上，新建一个引用类型实例，得到的变量值对应的是该实例的内存分配地址。值类型之间的赋值，原变量的变化不会引起新变量的变化；引用类型之间赋值，二者就指向了同一块内存区域，无论哪一个变量发生了变化，二者都会改变。
装箱和拆箱操作：装箱就是值类型到引用类型的转化过程。将一个值类型变量装箱成一个引用类型变量，首先会在托管堆上为新的引用类型变量分配内存空间，然后将值类型变量拷贝到托管堆上新分配的对象内存中，最后返回新分配的对象内存地址。装箱操作是可逆的，所以还有拆箱操作。拆箱操作获取指向对象中包含值类型部分的指针，然后由程序员手动将其对应的值拷贝给值类型变量。

C#中的委托

C# 中的委托（Delegate）类似于 C 或 C++ 中函数的指针。委托（Delegate） 是存有对某个方法的引用的一种引用类型变量。引用可在运行时被改变。委托声明决定了可由该委托引用的方法。委托可指向一个与其具有相同标签的方法

前言

在使用C++中的过程中，由于其动态分配内存的特殊机制，很容易出现new出一块内存分配，但程序还没执行到delete就跳转或返回的情况。如果new出来的内存没有及时得到释放，就会造成内存泄漏。使用智能指针就能在很大程度上避免这个问题，从定义上来看智能指针就是一个类，当超出类的作用域范围时就会调用析构函数释放资源，这也就是智能指针智能的地方。

智能指针介绍

auto_ptr（C++11已弃用）

在C++11的标准下，原本可能造成不安全的智能指针auto_ptr被弃用了。由于auto_ptr可以直接赋值给另外一个auto_ptr，所以当在函数间传递auto_ptr时，哪怕只是一次值传递，auto_ptr关联的内存块所有权也会传递到值传递参数的临时变量中，然后随着函数执行完成，临时变量离开作用域对应内存随之释放。这时候程序再访问原来的auto_ptr就会导致程序崩溃。

数据结构之链表

发表于 2019-03-10 | 分类于 基础

前言

程序=数据结构+算法，所以对于一个计算机专业的学生来说，数据结构的重要性不言而喻。
数据结构，从定义上说是计算机存储、组织数据的方式，不同的数据结构其操作效率截然不同。熟悉每一种数据结构的特性，在实际项目中我们就能择情选择最优的数据结构进行使用。

表

表，又称线性表，是最基础的一种数据结构。逻辑上表中元素为线性关系，表描述的是元素之间一对一的关系，表中元素之间具有连续性。表的实现方式有两种，一种是顺序表，另一种就是这里主要要讲到的链表。
顺序表是用一组连续地址的存储单元依次存储线性表中数据元素的结构，在计算机内部以物理相邻的位置表示线性表中元素之间的逻辑相邻，一般用数组的方式实现，对于数组后面会单独再整理一篇进行介绍。
使用顺序表需要预先分配表的大小，超过原有大小后需要重新分配连续的空间。而链表不用预先分配大小可以实现动态的存储，但存储元素时需要存储额外的信息以实现表的逻辑关系。

链表

链表使用一组任意的存储单元存储线性表中的数据元素。元素之间的关系依赖于元素之间的联系。每个数据对象实例存储在独立的单元或结点中进行描述，但每个结点中都包含了该结点相关的其他结点的位置信息。
链表分为单链表、双链表和循环链表。

Unity脚本常用API整理

发表于 2019-03-09 | 分类于 unity

MonoBehaviour

MonoBehaviour是unity中所有组件都要继承的基类，只有继承了MonoBehaviour才能挂载到游戏场景中的游戏对象上。可以说，MonoBehaviour为unity的组件式开发提供了基础。但是在实际项目过程中我们不能一味的继承它，因为它会破坏我们原有的代码架构，并且会增大代码的维护难度，一般来说在这些情况下我们才选择继承MonoBehaviour：

  1. 协助渲染的脚本，像是动画控制、场景效果
  2. 需要物理碰撞与触发(OnCollision与OnTrigger)
  3. 需要使用coroutine的脚本(有時可以通过额外的GameObject来解决)
  4. 其他与游戏主要逻辑无关的支援脚本，例如：FPS显示、debug界面

MonoBehaviour的生命周期：

  • Awake：当一个脚本被实例化时，Awake 被调用。我们大多在这个类中完成成员变量的初始化。

  • Start：仅在 Update 函数第一次被调用前调用。因为它是在 Awake 之后被调用的，我们可以把一些需要依赖 Awake 的变量放在Start里面初始化。 同时我们还大多在这个类中执行 StartCoroutine 进行一些协程的触发。

  • Update：当开始播放游戏帧时（此时，GameObject 已实例化完毕），其 Update 在每一帧被调用。
