Unity脚本常用API整理

发表于 2019-03-09 | 分类于 unity

MonoBehaviour

MonoBehaviour是unity中所有组件都要继承的基类，只有继承了MonoBehaviour才能挂载到游戏场景中的游戏对象上。可以说，MonoBehaviour为unity的组件式开发提供了基础。但是在实际项目过程中我们不能一味的继承它，因为它会破坏我们原有的代码架构，并且会增大代码的维护难度，一般来说在这些情况下我们才选择继承MonoBehaviour：

  1. 协助渲染的脚本，像是动画控制、场景效果
  2. 需要物理碰撞与触发(OnCollision与OnTrigger)
  3. 需要使用coroutine的脚本(有時可以通过额外的GameObject来解决)
  4. 其他与游戏主要逻辑无关的支援脚本，例如：FPS显示、debug界面

MonoBehaviour的生命周期：

  • Awake：当一个脚本被实例化时，Awake 被调用。我们大多在这个类中完成成员变量的初始化。

  • Start：仅在 Update 函数第一次被调用前调用。因为它是在 Awake 之后被调用的，我们可以把一些需要依赖 Awake 的变量放在Start里面初始化。 同时我们还大多在这个类中执行 StartCoroutine 进行一些协程的触发。

  • Update：当开始播放游戏帧时（此时，GameObject 已实例化完毕），其 Update 在每一帧被调用。

  • LateUpdate：LateUpdate 是在所有 Update 函数调用后被调用。

  • FixedUpdate：当 MonoBehaviour 启用时，其 FixedUpdate 在每一固定帧被调用。

  • OnEnable：当对象变为可用或激活状态时此函数被调用。

  • OnDisable：当对象变为不可用或非激活状态时此函数被调用。

  • OnDestroy：当 MonoBehaviour 将被销毁时，这个函数被调用。

  • Invoke：在 time 秒之后，调用methodName方法。
    void Invoke (methodName : string, time : float) :

  • InvokeRepeating：从第一次调用开始,每隔repeatRate时间调用一次.
    void InvokeRepeating (methodName : string, time : float, repeatRate : float) :

  • CancelInvoke:取消这个MonoBehaviour上的所有调用Invoke。
    void CancelInvoke () :

  • IsInvoking:某指定函数是否在等候调用。
    bool IsInvoking (methodName : string) :

  • StartCoroutine:开启协程。一个协同程序在执行过程中,可以在任意位置使用 yield 语句。yield 的返回值控制何时恢复协同程序向下执行。协同程序在对象自有帧执行过程中堪称优秀。协同程序在性能上没有更多的开销。StartCoroutine函数是立刻返回的,但是yield可以延迟结果(yield return new WaitForSeconds（秒数）)。直到协同程序执行完毕。
    Coroutine StartCoroutine (routine : IEnumerator) :

  • StopCoroutine / StopAllCoroutines：停止协程/停止所有运行中的协程。

  • OnMouseEnter／OnMouseOver／OnMouseExit／OnMouseDown／OnMouseUp／OnMouseDrag：
    当鼠标进入／悬浮／移出／点击／释放／拖拽GUIElement(GUI元素)或Collider(碰撞体)中时调用

  • OnTriggerEnter／OnTriggerExit／OnTriggerStay：当Collider(碰撞体)进入／退出／停留在trigger(触发器)时调用OnTriggerEnter。OnTriggerStay 将会在每一帧被调用。

  • OnCollisionEnter／OnCollisionExit／OnCollisionStay：当此collider/rigidbody触发另一个rigidbody/collider时，被调用。OnCollisionStay 将会在每一帧被调用。

Time

Time是游戏中经常要使用到的一个类，在实现基本的移动时通常会乘上一个Time.deltatime，在游戏中需要有时间间隔设定的地方也经常要使用Time.time来获取当前游戏时间。这里整理了一下Time类的一些常用属性。

  • Time.time：（只读）表示从游戏开发到现在的时间，会随着游戏的暂停而停止计算。

  • Time.timeSinceLevelLoad：（只读）表示从当前Scene开始到目前为止的时间，也会随着暂停操作而停止。

  • Time.deltaTime：（只读）表示从上一帧到当前帧时间，以秒为单位。

  • Time.fixedTime：（只读）表示以秒计游戏开始的时间，固定时间以定期间隔更新（相当于fixedDeltaTime）直到达到time属性。

  • Time.fixedDeltaTime：表示以秒计间隔，在物理和其他固定帧率进行更新，在Edit->ProjectSettings->Time的Fixed Timestep可以自行设置。

  • Time.maximumDeltaTime：一帧能获得的最长时间。物理和其他固定帧速率更新(类似MonoBehaviour FixedUpdate)。

  • Time.SmoothDeltaTime：（只读）表示一个平稳的deltaTime，根据前
    N帧的时间加权平均的值。

  • Time.timeScale：时间缩放，默认值为1。若设置<1，表示时间减慢；若设置>1,表示时间加快；若设置=0，则游戏暂停。可以用来加速、减速和暂停游戏，非常有用。

  • Time.frameCount：（只读）总帧数

  • Time.realtimeSinceStartup：（只读）表示自游戏开始后的总时间，即使暂停也会不断的增加。

  • Time.captureFramerate：表示设置每秒的帧率，然后不考虑真实时间。

  • Time.unscaledDeltaTime：（只读）不考虑timescale时候与deltaTime相同，若timescale被设置，则无效。

  • Time.unscaledTime：（只读）不考虑timescale时候与time相同，若timescale被设置，则无效。

GameObject&Compoment

GameObject，游戏对象，在unity中层级列表中的每个物体都是一个游戏对象,component是游戏对象上的组件。在进行开发时，会有很多要与游戏对象和组件打交道的操作，常用的有下面这些：
创建游戏对象

  • GameObject.Instantiate()：
    public static Object Instantiate(Object original);
    public static Object Instantiate(Object original, Transform parent);
    根据已经创建好的prefeb生成游戏对象，一共五个重载方法对应设置生成游戏对象的位置，方向，父对象

  • GameObject.CreatPrimitive(PrimitiveType.Cube)：根据Type后的固定类型创建游戏对象

实例化的游戏对象可通过AddComponent<>()方法添加组件或自定义的脚本，通过GetComponent<>()获取组件或自定义的脚本，可通过.tag设置游戏物体的tag属性。
activeInHierarchy表示游戏物体在层级视图中的激活状态，与activeSelf属性不同的是在父物体禁用时子物体的activeInHierarchy也为false而activeSelf为true。
物体的激活可通过SetActive(false/true)方法设定。

Static Methods

  • Destroy:销毁游戏物体或组件；

  • DestroyImmediate：立即销毁游戏物体或组件(建议使用Destroy)；

  • DontDestroyOnLoad：进行场景切换时不销毁一个游戏物体，可用于设置不同场景的共享物体；

  • FindObjectOfType：根据类型在场景中找到符合条件的第一个组件（不查找未激活游戏物体）；

  • FindObjectsOfType：根据类型在场景中找到符合条件所有组件，返回值是一个数组（不查找未激活游戏物体）；

  • Find：通过名字查找一个游戏对象；

  • FindWithTag：通过查找游戏对象的tag标签名查找游戏对象；

  • FindGameObjectsWithTag：通过查找游戏对象的tag标签名查找多个游戏对象，返回值是数组；

Public Methods

  • BroadcastMessage：对一个游戏对象及其子对象广播消息（根据方法名调用接收对象的方法，需要接收的对象脚本中有这个方法），可设定传递参数及是否需要接收对象；

  • SendMessage：只对一个游戏对象发送消息；

  • SendMessageUpwards：对一个游戏对象及其父对象发送消息；

  • GetComponent：查找游戏对象的一个组件；

  • GetComponents:查找一个游戏对象的同类型所有组件；

  • GetComponentInChildren：查找游戏对象及其子对象的一个组件；

  • GetComponentsInChildren: 查找游戏对象及其子对象的同类型所有组件；

  • GetComponentInParent: 查找游戏对象及其父对象的一个组件；

  • GetComponentsInParent: 查找游戏对象及其父对象的同类型所有组件；

更多详见官方文档

Transform

Transform是所有游戏对象都有的一个组件，主要表示物体在三维空间的位置坐标，旋转方向以及缩放比例。通过更改它的属性可以控制物体的移动，旋转和大小变换等操作。

Properties

  • chlidCount:返回对应transform的子对象个数

  • eulerAngles:返回一个旋转对应的欧拉角度数,unity中欧拉旋转顺序是z、x、y。

  • localEulerAngles:本地坐标系下的旋转欧拉角度数（本地坐标即相对父对象坐标系的变换量）

  • localPosition:本地坐标系的位置值（相对位置），即相对父对象的位置坐标

  • localRotation:本地坐标系的旋转，相对父对象的旋转

  • localScale:相对父对象的缩放比例大小

  • parent：父对象的transform

  • position：世界坐标系下的位置（绝对位置）

  • rotation：世界坐标系下的旋转，以四元数的形式存储

  • foward：z轴对应的正方向

  • right:x轴对应的正方向

  • up:y轴对应的正方向

  • localToWorldMatrix:矩阵变换的点从自身坐标转为世界坐标

  • worldTolocalMatrix:矩阵变换的点从世界坐标转为自身坐标

Public Methods

  • DetachChlidren:解除所有子对象与自身的父子关系

  • Find:找到一个子对象transform并返回它

  • GetChild:通过index值返回子对象transform

  • LookAt:改变当前transform的旋转使得它的foward方向指向target的当前位置

  • Rotate:应用一个欧拉角的旋转角度，eulerAngles.z度围绕z轴，eulerAngles.x度围绕x轴，eulerAngles.y度围绕y轴；如果相对于留空或者设置为Space.Self 旋转角度被应用围绕变换的自身轴。（当在场景视图选择物体时，x、y和z轴显示）如果相对于 Space.World 旋转角度被应用围绕世界的x、y、z轴。

  • RotatorAround:按照angle度数围绕世界坐标对应的点的轴旋转物体

  • SetParent：设置当前对象的父对象

  • TransformPoint:变换位置，将本地坐标变换为世界坐标

  • Translate:向指定方向和距离移动游戏对象

更多详见官方文档

Vector2/Vector3

unity中用Vector来定义向量，其中常用的为二维向量vector2和三维向量vector3，vector类型在初始化时要整体赋值。

Static Properties

  • down:坐标（0，-1）/（0，-1,0）
  • left:坐标（-1,0）/（-1,0,0）
  • right:坐标（1,0）/（1,0,0）
  • up:坐标（0,1）/（0,1，0）
  • negativeInfinity:坐标（-∞，-∞）/（-∞，-∞，-∞）
  • positiveInfinity:坐标（+∞，+∞）/（+∞，+∞，+∞）
  • one:坐标（1,1）/（1,1,1）
  • zero:坐标（0,0）/（0,0,0）
  • forward:坐标（0,0,1）
  • back:坐标(0,0,-1)

Properties

  • magnitude：返回一个向量的长度；
  • normalized:返回将一个向量单位化后的向量（长度为1），对原本的坐标不产生影响；
  • sqrmagnitude:返回一个向量长度的平方（求向量长度有开方操作更耗费性能）；

Static Methods

  • Angle：返回两个向量的夹角；

  • ClampMagnitude：ClampMagnitude(Vector2 a, float maxLength)，将一个向量的长度限制在自身a到maxLength之间；

  • Distance：Distance(Vector2 a, Vector2 b)，返回向量a-b的长度；

  • Dot:返回两个向量的点乘；

  • Lerp：同Mathf类的插值运算，在两个坐标范围内将坐标按比例返回一个坐标；

  • LerpUnclamped：LerpUnclamped(Vector2 a, Vector2 b, float t),当比例系数t大于1时最后返回的向量长度可大于b;

  • Max:返回向量中长度最大的一个向量；

  • Min:返回向量中长度最小的一个向量；

  • MoveTowards: MoveTowards(Vector2 current, Vector2 target, float maxDistanceDelta)，向量current向target坐标以不超过maxDistanceDelta的速度匀速接近；

  • Cross:返回两个向量的叉乘；

  • Normalize：将自身单位化(方向不变，长度为1)；

Input

Input用于获取用户的各种输入操作，常用的如下：

Static Methods

  • GetKey：键盘上的一个按键持续按下触发；

  • GetKeyDown：键盘的一个按键按下触发；

  • GetKeyUp：键盘上的一个按键抬起触发；

  • GetMouseButton：鼠标按键持续按下时执行，鼠标左键0，中键（滑轮）2，右键1；

  • GetMouseButtonDown：鼠标按下时触发；

  • GetMouseButtonUp：鼠标按键抬起时触发；

  • GetButton：虚拟按键持续按下时触发；

  • GetButtonDown：虚拟按键按下时触发；

  • GetButtonUp：虚拟按键抬起时触发；

  • GetAxis：虚拟按键中的轴（Horizontal控制水平方向移动和旋转，Vertical控制前后方向的移动）中的按键按下时执行，返回一个由0到1再由1到0渐变的值，一般用于控制加速类运动；

  • GetAxisRaw：虚拟按键中的轴中的按键按下时执行，返回值为固定的1,-1，控制匀速类运动；

注：虚拟按键和轴均在InputManager里面设定
